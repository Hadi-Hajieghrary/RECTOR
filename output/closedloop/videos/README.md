# Enhanced Closed-Loop BEV Rollout Videos

This directory contains 55 bird's-eye-view (BEV) simulation videos demonstrating RECTOR's trajectory selection in diverse real-world driving scenarios from the Waymo Open Motion Dataset. Each video shows a full 8-second closed-loop rollout with dual-panel visualization: the driving scene on the left and rule violation metrics on the right.

These videos serve as **supplementary material** for the paper, providing visual evidence that RECTOR's lexicographic selection produces rule-compliant trajectories across a representative range of driving maneuvers.

---

## Visualization Format

Each video frame consists of two panels:

### Left Panel: Bird's-Eye-View Scene

- **Ego vehicle** (green rectangle): The autonomous vehicle being controlled
- **Other agents** (blue rectangles): Surrounding vehicles, pedestrians, and cyclists from the WOMD scenario, rendered with oriented bounding boxes
- **Road geometry**: Lane centerlines, road edges, crosswalks, and stop signs rendered from up to 20,000 roadgraph points per scenario
- **6 candidate trajectories** (colored lines ahead of ego): All trajectory modes generated by RECTOR, each rendered as a thin colored line projecting forward from the ego's current position
- **Selected trajectory** (thick line with arrow): The candidate chosen by the lexicographic selector, highlighted with increased line width and a directional arrowhead
- **Executed trail** (solid green): The path the ego has already driven during the simulation
- **Logged trail** (dashed cyan): The ground-truth human driver path for comparison

### Right Panel: Rule Violation Metrics

- **Grouped bar chart**: For each of the 6 candidate trajectories, four bars represent the aggregated violation cost across the four rule tiers:
  - **Safety** (red): Collision avoidance, VRU clearance, safe following distance
  - **Legal** (orange): Red lights, stop signs, speed limits, right-of-way
  - **Road** (yellow): Drivable surface, lane departure
  - **Comfort** (blue): Smooth acceleration, braking, steering, following distance
- **Selected candidate**: Indicated by a bold border around its bars
- **Confidence scores**: Each candidate's model confidence shown in the legend

This side-by-side layout makes it immediately visible *why* RECTOR selects a particular trajectory — the selected mode consistently has the lowest violation costs, prioritizing safety above all other tiers.

## Scenario Mining Methodology

Scenarios were selected to demonstrate RECTOR across **diverse driving maneuvers**, not just the most dramatic or highest-scoring situations. The mining algorithm classifies each WOMD scenario's ego trajectory into one of five maneuver categories and then samples proportionally from each category.

### Maneuver Classification

Each scenario's ego ground-truth trajectory is analyzed for:

| Feature | Computation | Purpose |
|---------|-------------|---------|
| `net_heading_change` | End-to-end heading difference (unwrapped) | Distinguishes turns from lane changes |
| `total_heading_change` | Cumulative absolute heading deltas | Measures total turning effort |
| `lateral_displacement` | Maximum perpendicular offset from initial heading | Detects lateral maneuvers |
| `lat_reversals` | Direction changes in smoothed lateral profile | Identifies overtaking/weaving |
| `speed_variance` | Standard deviation of speed profile | Detects stop-and-go |
| `transitions` | Speed zero-crossing count | Identifies multi-phase maneuvers |
| `infra_score` | Nearby crosswalks + stop signs + agent count | Measures scene complexity |

### Category Definitions

| Category | Criteria | Target Allocation |
|----------|----------|-------------------|
| **TURN** | total_heading > 1.5 rad AND \|net_heading\| > 0.8 rad | 30% |
| **LANE_CHANGE** | lateral_disp > 2.0 m AND total_heading < 1.5 rad | 30% |
| **EXIT_RAMP** | 0.3 < \|net_heading\| < 1.5 rad AND lateral_disp > 1.0 m | 20% |
| **COMPLEX** | transitions >= 2 OR (speed_var > 3.0 AND lat_reversals >= 1) | 15% |
| **OTHER** | None of the above | 5% |

### Mining Statistics

From scanning 3,600 candidate scenarios (60 shards x 60 scenarios/shard):
- 2,202 had valid ego trajectories with sufficient motion
- Distribution found: 248 TURN, 305 LANE_CHANGE, 37 EXIT_RAMP, 384 COMPLEX, 1,228 OTHER

### Final Distribution in This Directory

| Category | Count | Scenario Examples |
|----------|-------|-------------------|
| TURN | 16 | Intersections, U-turns, protected/unprotected lefts |
| LANE_CHANGE | 18 | Highway merges, lateral shifts, overtaking |
| EXIT_RAMP | 11 | Highway exits, moderate turns with lateral offset |
| COMPLEX | 8 | Stop-and-go traffic, multi-phase maneuvers |
| OTHER | 2 | Straight cruising, highway driving |

## Closed-Loop Simulation Details

Each video shows an 80-step (8-second) closed-loop simulation using the Waymax simulator:

| Parameter | Value |
|-----------|-------|
| Simulation frequency | 10 Hz |
| Total steps | 80 (8 seconds) |
| Re-planning interval | Every 20 steps (2 seconds) |
| Candidate trajectories | 6 modes per re-plan |
| Prediction horizon | 5 seconds (50 timesteps) |
| Dynamics model | DeltaGlobal |
| Other agents | Log-replay (ground-truth playback) |
| Overlap detection | OBB collision checking at every step |
| Roadgraph points | Up to 20,000 per scenario |

### Trajectory Generation

The `DiverseMockGenerator` creates 6 visually distinct trajectory candidates:
- **Mode 0**: Clean logged (ground-truth) trajectory
- **Modes 1-5**: Perturbations with lateral offsets, longitudinal shifts, and Gaussian noise
- Each mode receives synthetic rule violation costs that simulate realistic violations (safety violations for aggressive modes, comfort violations for jerky modes)

### Coordinate Transform Pipeline

Trajectories follow this coordinate transform chain:
1. **Ego-local normalized** [K, T, 4]: Model output (x, y, vx, vy) divided by TRAJECTORY_SCALE=50
2. **Denormalize**: Multiply by 50 to get meters
3. **Rotate**: Apply ego heading rotation matrix to convert from ego frame to world frame
4. **Translate**: Add ego position to get global coordinates

## File Naming Convention

```
scenario_{INDEX}_{SCENARIO_ID}.mp4
```

- `INDEX`: Zero-padded sequential index (000-054)
- `SCENARIO_ID`: First 16 hex characters of the WOMD scenario UUID

## Produced By

**Script:** [`scripts/simulation_engine/viz/enhanced_bev_rollout.py`](../../../scripts/simulation_engine/viz/enhanced_bev_rollout.py)

**Command:**
```bash
cd /workspace
python scripts/simulation_engine/viz/enhanced_bev_rollout.py \
    --target_count 55 \
    --shards_to_scan 60 \
    --scenarios_per_shard 60
```

**Dependencies:**
- Waymax simulator (JAX-based)
- WOMD validation TFRecords at `data/WOMD/datasets/.../validation_interactive/`
- matplotlib, ffmpeg (video encoding)

**Runtime:** ~56 minutes for 55 videos on a single GPU
